Overview

Warrior Coins is a full economy plugin with timed passive income, a daily login bonus, player bounties, a player shop, and an in-game shop admin editor. It is built for Rust servers running Oxide/uMod and stores all economy data in local plugin data files.

Requirements
  • Rust server with Oxide/uMod installed.
  • No external plugin dependencies are required.
  • Players must have normal chat command access to use economy and bounty commands.
What the Plugin Handles
  • Timed earnings: Grants coins every configured interval based on highest earn permission tier.
  • Daily login bonus: Optional once-per-24-hour coin reward when players join.
  • Bounties: Players place/cancel bounties, and killers automatically receive payouts.
  • Economy transfers: Players can pay each other directly using chat commands.
  • Shop system: In-game purchasable items/commands with per-player cooldown support.
  • HUD: Optional top-right WC balance display with per-player toggle.

Player Chat Commands

/wc pay <player> <amount>
/wctoggle
/bounty set <player> <amount>
/bounty cancel <player>
/bounty mine
/bounty onme
/shop
  • /wc - Shows your current Warrior Coins balance.
  • /wc top - Shows top 10 balances.
  • /wc pay - Sends WC to another player.
  • /wctoggle - Turns the WC HUD on/off.
  • /bounty ... - Full bounty placement, cancel, and status controls.
  • /shop - Opens the player shop UI.
Admin Commands
/wc set <player> <amount>
/wc give <player> <amount>
/wc take <player> <amount>
/wc wipe
/wcadmin
  • /wc set - Set a player balance to an exact value.
  • /wc give - Add WC to a player.
  • /wc take - Remove WC from a player.
  • /wc wipe - Wipes balances, bounties, and daily bonus tracking.
  • /wcadmin - Opens the in-game shop admin editor UI.

Admin access requires either warriorcoins.admin or Rust auth level 2.

Shop Delivery Types
  • item - Gives item shortname + amount (optional workshop skin ID supported).
  • command - Runs a configured console command after purchase.
  • Each shop listing supports cost, cooldown, category, icon URL, and enabled state.

Core Config Defaults
CurrencyNameShort: WC CurrencyNameLong: Warrior Coins EarnIntervalSeconds: 300 EnableEarnMessages: false DailyLoginBonusEnabled: true DailyLoginBonusAmount: 25 ProcessDeathHook: true ShopGridColumns: 6 ShopGridRows: 3

Timed earnings and daily bonus values can be tuned to fit your server economy. Shop branding, logo, and UI font are also configurable.

Quick Setup for Server Owners
  1. Install WarriorCoins.cs in your plugins folder and reload it.
  2. Assign earn permissions to your player/VIP groups.
  3. Give admin staff warriorcoins.admin (or auth level 2).
  4. Use /wcadmin in-game to create shop listings.
  5. Optionally tune config values for interval speed, bonus amount, and shop branding.
Notes & Troubleshooting
  • Players not earning coins: Ensure they have at least one warriorcoins.earn.* permission.
  • No bounty payout: Verify ProcessDeathHook is enabled and kills are player-vs-player.
  • Shop purchase issues: Validate shortnames, command syntax, and item cooldown settings.
  • Need a full reset: Use /wc wipe (admin only).
