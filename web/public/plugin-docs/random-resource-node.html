Random Resource Node

Plugin Documentation for Rust (Oxide/uMod)

Version 1.3.9 · Author: Rust Server Solutions

Created for server owners - complete setup, permissions, commands, and configuration reference
Overview

Random Resource Node spawns a single infinite ore vein at a random location inside a configured circle. It can run as a timed event (spawns every interval, despawns after a duration), or as an always-on system (a vein is always present and relocates when the duration ends).

Requirements
  • Rust server with Oxide/uMod installed.
  • No external plugin dependencies are required.
  • To use admin commands, admins need randomresourcenode.admin (or Rust auth level 2).
What the Plugin Handles
  • Random vein spawning: Selects a random ore prefab from your configured list, then spawns it at a safe surface location.
  • Infinite mining: Automatically refills the node’s ResourceDispenser amounts after gathers so it never depletes.
  • Optional announcements: Broadcasts a customizable message when a vein spawns, including map grid and duration.
  • World map markers: Optional red radius circle marker + vending marker at the node position.
  • Discovery flare: Optional “champagne boomer” firework near the node to help players find it.
  • HQM bonus (metal only): Optional extra hq.metal.ore granted while mining the active metal vein.
Permissions
Permission Purpose Notes
randomresourcenode.admin Access to /rrn admin chat command Rust auth level 2 is also allowed
Player Chat Commands

This plugin does not include player-facing commands by default (only admin tools).

Admin Commands
/rrn
/rrn here
/rrn respawn
/rrn pos
/rrn griddebug
  • /rrn - Shows help and the current running mode.
  • /rrn here - Sets the spawn center (CenterX/CenterZ) to your current position and saves the config.
  • /rrn respawn - Forces a spawn immediately (in event mode, timers are reset).
  • /rrn pos - Shows the active node world position and approximate grid.
  • /rrn griddebug - Prints grid math diagnostics (extent, cell size, indices) for you and the node.

Quick Setup for Server Owners
  1. Install RandomResourceNode.cs in your Oxide/uMod plugins folder and reload it.
  2. Run /rrn here in-game (admin) to set the center point.
  3. Adjust SpawnRadiusMeters (and MinDistanceFromPlayers if needed) to fit your server.
  4. Choose a mode:
    • Event mode: Set EventIntervalMinutes to the minutes between spawn starts (e.g. 60).
    • Always-on mode: Set EventIntervalMinutes to 0 so a node is always present.
  5. Optionally customize the announcement message (AnnouncementFormat) and map markers.
Notes & Troubleshooting
  • Nothing spawns: Make sure CenterX/CenterZ are set (use /rrn here) and that SpawnRadiusMeters isn’t placing attempts outside valid terrain.
  • Repeated “spawn failed” warnings: The plugin retries after 60 seconds when it can’t find a valid spot or can’t create the entity. Increase MaxSpawnAttempts, reduce MinDistanceFromPlayers, or reduce SpawnRadiusMeters.
  • Node spawns under objects / clipped: The plugin raycasts down from above terrain to find an open surface. Tight maps may need tuning MaxMetresBelowHeightmapForSurface.
  • Map markers not visible: Ensure DisableWorldMapMarkers is false. Some Rust builds change prefab paths; if you see warnings about marker prefabs, it’s usually a Rust update compatibility issue.
  • Discovery flare not working: Set DisableDiscoveryFlare to true if your server build can’t spawn/ignite the champagne boomer prefab.
  • HQM bonus not granted: Bonus only applies when mining the active metal vein (not stone/sulfur). Adjust MetalVeinHqmPerMetalOreRatio and min/max per gather.
  • Grid looks “off” compared to your overlay: Set AnnouncementGridCellMeters to match your overlay cell size (0 uses ~146.3m cells).
