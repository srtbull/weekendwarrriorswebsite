Advanced Refinery

Plugin Documentation for Rust (Oxide/uMod)

Version 6.0.4 · Author: Banshee

Converts a small furnace into a refinery that accepts wood, low grade fuel, and diesel; outputs diesel from wood + LGF on a timer.
Overview

Advanced Refinery unlocks a deployed small furnace so it no longer uses the default “cookables only” filter. Staff or trusted players with permission aim at a small furnace and run /refineryadd to register it. Registered furnaces get a distinct skin, six inventory slots, and a processing loop: on each tick, if there is at least 1 wood and enough low grade fuel (see config), the plugin consumes that fuel and wood and creates diesel_barrel. If inputs are missing, the furnace is turned off until the next successful cycle.

Requirements
  • Rust server with Oxide/uMod.
  • Target entity must be the small furnace prefab (furnace.prefab).
  • No other plugin dependencies.
Permissions
Permission Purpose
advancedrefinery.use Allows /refineryadd to convert a looked-at small furnace.
Chat command
/refineryadd
  • Stand close and look at a small furnace (raycast ~5 m).
  • Without permission: chat message “No permission.”
  • If not aiming at a small furnace: “Look at a small furnace.”
  • On success: furnace is configured, its network ID is saved, and you see a confirmation message.
Config
LowGradePerDiesel: 100
Multiplier: 1
TickRate: 2
  • LowGradePerDiesel — Low grade fuel consumed per diesel craft (float; cast to int when used).
  • Multiplier — Diesel barrels created per successful cycle (floored; minimum 1).
  • TickRate — Seconds between processing passes for all registered refineries.
Data file

Persisted under Oxide data as AdvancedRefinery.json (plugin name). Stores a set of furnace entity IDs so conversions survive restarts. Invalid or removed entities are skipped on tick; cleanup of stale IDs is not automatic beyond natural culling when entities are gone.

Technical notes
  • CanAcceptItem only allows wood, lowgradefuel, and diesel_barrel into registered refinery containers.
  • Inventory is forced to generic contents, filter callbacks cleared, capacity set to 6, then networked.
  • If the new diesel item cannot fit the container, it is dropped at the furnace drop position.
